# 更新・スケジューリング仕様（v1）

本章はSYS-REQ-041を満たすための設計方針を記載する（要求そのものは要求章に定義）。

## 方針
- `AlarmManager` を用いて、更新完了後に次回更新を “約1分後” にスケジュールする（自己再スケジュール）
- exact alarm が許可されない端末向けに、フォールバックを用意する
  - setAndAllowWhileIdle 等
  - Widget手動更新（SYS-REQ-044）で最低限のUXを担保

## 期待動作
- 省電力状態では更新間隔が伸びる可能性がある
- Widgetに最終更新時刻（SYS-REQ-040）を表示し、遅延をユーザーが把握可能にする
